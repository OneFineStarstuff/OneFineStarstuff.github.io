6Executive Decisions Required ACTION NEEDED
Budget Correction Required — Two Architecture Options
The 15% budget overrun ($240K) is driven by LLM token intensity. Both options below use RAG architecture levers — no scope reduction required. ESC decision deadline: Feb 14, 2026.
Option A — Aggressive Optimization
Context window resize + top-k reduction + model tiering
Projected Savings
$175K (73% of overrun)
Implementation Time
3 weeks (Weeks 17–19)
Accuracy Risk
-0.8 to -1.5 pp (reversible)
Residual Overrun
$65K (4.1%)
Technical Levers:
  • Context window: 16K → 8K tokens. Reduce injected context per query by 50%. Requires improved chunking strategy (already in sprint backlog) and semantic chunk scoring. Cuts avg tokens/query from 4,200 to ~2,400.
  • Retrieval top-k: 10 → 5. Fewer retrieved passages per query. Offset by Cohere reranker (completing Week 17) which improves precision@5 to match current recall@10. Net accuracy impact: neutral to +0.5 pp.
  • Model mix: GPT-4 68% → 35%. Deploy query-complexity classifier to route simple queries (factual lookups, single-doc) to GPT-3.5-Turbo. Reserve GPT-4 for multi-hop, legal reasoning, ambiguous queries only. Estimated mix: 35% GPT-4 / 65% GPT-3.5-Turbo (matching original plan).
Pros: Highest savings; aligns to original budget model; no scope reduction; reranker compensates for top-k reduction.
Cons: Temporary accuracy dip (0.8–1.5 pp) during Week 17 transition while reranker stabilizes; requires 2 engineer-weeks of query router development; Golden Set may temporarily drop to 86% before recovering.
Option B — Moderate Optimization + Model Diversification
Context window resize + multi-model routing (GPT-4o-mini / Claude 3 Haiku)
Projected Savings
$120K (50% of overrun)
Implementation Time
4 weeks (Weeks 17–20)
Accuracy Risk
-0.3 to -0.8 pp (lower risk)
Residual Overrun
$120K (7.5%)
Technical Levers:
  • Context window: 16K → 12K tokens. Conservative 25% reduction. Lower risk to long-context legal queries. Cuts avg tokens/query from 4,200 to ~3,200.
  • Multi-model routing via LiteLLM. Introduce GPT-4o-mini ($0.00015/1K) and Claude 3 Haiku ($0.00025/1K) for Tier-1 queries. ~40% of query volume routes to economy models. Reduces vendor concentration risk simultaneously.
  • No top-k change. Maintains current retrieval depth. Lower precision risk.
Pros: Lower accuracy risk; diversifies vendor dependency; smaller engineering lift (LiteLLM already in staging); sets up multi-model architecture for long-term cost optimization.
Cons: Saves only 50% of overrun ($120K), leaving 7.5% breach above budget; 4-week timeline delays full savings realization; multi-model evaluation adds QA complexity; does not address top-k inefficiency.
PMO Recommendation: Option A
Option A recovers 73% of the overrun and restores the GPT-4/3.5 mix to the originally approved ratio. The temporary accuracy dip is manageable given that GA is Week 22 and the reranker (completing Week 17) is expected to more than compensate. Combined with the Cohere reranker, Option A may yield a net accuracy improvement by Week 19 despite using less context. Option B is the recommended fallback if the query-complexity router proves infeasible within 2 engineering-weeks.
